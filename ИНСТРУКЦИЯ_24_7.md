## 🔧 РУЧНАЯ НАСТРОЙКА (если автоматическая не сработала)

### 1. Системный сервис

```bash
# Копируем файл сервиса
sudo cp systemd/telegram-bot.service /etc/systemd/system/

# Редактируем пути в файле (замените /home/ubuntu/telegram-bot на ваш путь)
sudo nano /etc/systemd/system/telegram-bot.service

# Включаем сервис
sudo systemctl daemon-reload
sudo systemctl enable telegram-bot
sudo systemctl start telegram-bot
```

### 2. Автоматические бэкапы


## 🔒 БЕЗОПАСНОСТЬ

## 🚨 УСТРАНЕНИЕ НЕПОЛАДОК

### Бот не запускается

```bash
# Проверка логов
sudo journalctl -u telegram-bot -n 50

# Проверка конфигурации
cd /path/to/bot
source venv/bin/activate
python check_config.py

# Проверка токена
python -c "
from config import Config
config = Config()
print(f'Token: {config.BOT_TOKEN[:10]}...')
"
```

### Бот работает, но не отвечает

```bash
# Проверка подключения к Telegram
curl -X GET "https://api.telegram.org/bot$BOT_TOKEN/getMe"

# Проверка webhook
curl -X GET "https://api.telegram.org/bot$BOT_TOKEN/getWebhookInfo"

# Перезапуск бота
sudo systemctl restart telegram-bot
```

### Проблемы с базой данных

```bash
# Проверка файла базы данных
ls -la data/bot_database.db

# Проверка структуры
sqlite3 data/bot_database.db ".schema"

# Восстановление из бэкапа
cp backups/backup_daily_*/bot_database.db data/
```

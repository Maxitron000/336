# 🏥 Система мониторинга бота

## 📋 Обзор

Система мониторинга бота предоставляет главному админу полную информацию о состоянии и производительности бота, включая автоматические уведомления и ручные проверки.

## 🚀 Новые функции

### 1. Автоматические уведомления

#### Запуск/остановка бота
- **При запуске**: `🚀 Статус бота: STARTED` - отправляется главному админу
- **При остановке**: `🛑 Статус бота: STOPPED` - отправляется главному админу
- **При ошибках**: `🚨 Ошибка бота!` - отправляется всем админам

#### Ежедневные проверки
- **09:00** - Проверка здоровья бота
- **19:00** - Ежедневная сводка
- **20:30** - Напоминания отсутствующим

#### Еженедельные отчеты
- **Воскресенье 10:00** - Отчет о производительности

### 2. Команды для админов

#### `/health` - Проверка здоровья бота
```
🏥 Проверка здоровья бота

🤖 Бот: ✅ Имя_бота
🗄️ База данных: ✅
👥 Всего пользователей: 25
✅ Активных сегодня: 22
🕐 Время проверки: 09:00
📅 Дата: 15.12.2024

📊 Общий статус: ✅ Система работает нормально
```

#### `/performance` - Отчет производительности
```
📈 Отчет о производительности

🚀 Статус: 🟢 Отличная
📊 Процент присутствия: 88.0%
👥 Всего бойцов: 25
✅ Присутствуют: 22
❌ Отсутствуют: 3
🕐 Время: 10:00
📅 Дата: 15.12.2024
```

#### `/maintenance [тип] [длительность]` - Уведомление о ТО
```
🔧 Техническое обслуживание

🔧 Тип: scheduled
⏱️ Длительность: 30 минут
🕐 Время: 10:30
📅 Дата: 15.12.2024

ℹ️ Бот может быть временно недоступен
```

### 3. Админ-панель

#### 🏥 Мониторинг бота
- **Проверка здоровья** - отправляет отчет главному админу
- **Отчет производительности** - отправляет отчет главному админу
- **Статистика системы** - показывает локальную статистику
- **Детальная диагностика** - полная проверка всех компонентов
- **Автоматические проверки** - настройки расписания
- **История статусов** - последние события системы

#### 🔧 Техобслуживание
- **Плановое ТО** - регулярное обслуживание
- **Экстренное ТО** - срочные исправления
- **Обновление системы** - установка обновлений
- **Резервная копия** - создание бэкапов
- **Очистка данных** - удаление старых записей
- **Лог ТО** - история техобслуживания

## 📊 Типы уведомлений

### Статусы бота
- `started` - 🚀 Бот запущен
- `stopped` - 🛑 Бот остановлен
- `error` - ❌ Ошибка
- `warning` - ⚠️ Предупреждение
- `info` - ℹ️ Информация
- `success` - ✅ Успех
- `restart` - 🔄 Перезапуск
- `maintenance` - 🔧 Обслуживание

### Типы техобслуживания
- `scheduled` - 🔧 Плановое
- `emergency` - 🚨 Экстренное
- `update` - 🔄 Обновление
- `backup` - 💾 Резервная копия
- `cleanup` - 🧹 Очистка

## 🔧 Настройка

### Переменные окружения
```bash
# ID главного админа (первый в списке)
ADMIN_IDS=123456789,987654321

# Время автоматических проверок
DAILY_SUMMARY_TIME=19:00
REMINDERS_TIME=20:30
```

### Расписание автоматических проверок
- **09:00** - Ежедневная проверка здоровья
- **19:00** - Ежедневная сводка
- **20:30** - Напоминания
- **Воскресенье 10:00** - Еженедельный отчет производительности

## 📱 Примеры использования

### 1. Проверка здоровья через команду
```
/health
```
Результат: Отчет отправляется главному админу

### 2. Проверка через админ-панель
1. Открыть админ-панель
2. Выбрать "🏥 Мониторинг бота"
3. Нажать "🏥 Проверка здоровья"

### 3. Техобслуживание
1. Открыть админ-панель
2. Выбрать "🔧 Техобслуживание"
3. Выбрать тип ТО
4. Подтвердить выполнение

### 4. Уведомление о ТО через команду
```
/maintenance scheduled 30 минут
```

## 🛡️ Безопасность

- Уведомления отправляются только главному админу (первый в списке ADMIN_IDS)
- Уведомления об ошибках отправляются всем админам
- Все действия логируются в базу данных
- Подтверждение требуется для опасных операций

## 📈 Мониторинг производительности

### Критерии оценки
- **🟢 Отличная** (90%+ присутствия)
- **🟡 Хорошая** (70-89% присутствия)
- **🟠 Средняя** (50-69% присутствия)
- **🔴 Низкая** (<50% присутствия)

### Метрики
- Общее количество пользователей
- Количество присутствующих
- Процент присутствия
- Статус базы данных
- Количество последних событий

## 🔍 Диагностика

### Проверяемые компоненты
- Подключение к базе данных
- Доступность бота
- Чтение пользователей
- Чтение событий
- Системная информация

### Логирование
Все события мониторинга записываются в таблицу `events`:
- `maintenance` - события техобслуживания
- `health_check` - проверки здоровья
- `performance_report` - отчеты производительности
- `error` - ошибки системы

## 🚀 Развертывание

1. Обновить код с новыми функциями
2. Проверить настройки ADMIN_IDS
3. Перезапустить бота
4. Проверить отправку уведомления о запуске
5. Протестировать команды мониторинга

1. Проверить логи бота
2. Использовать команду `/health`
3. Проверить настройки уведомлений
4. Убедиться в корректности ADMIN_IDS

# 🛡️ Telegram Bot для военной части

## 📋 Описание

Telegram бот для учета личного состава воинской части с системой отметок прибытия и убытия.

## 🎯 Основная логика

### ✅ Прибыл
- **Всегда** означает прибытие **в расположение части**
- Статус: "В расположении"
- Локация: "В расположении"

### 🚪 Убыл  
- **Всегда** означает убытие **из расположения части**
- Статус: "Вне расположения"
- Локация: выбранная локация (больничный, отпуск, командировка и т.д.)

### 📍 По умолчанию
- Если боец не отмечался = **в расположении**
- Статус: "В расположении"

## 👥 Система ролей

### 👤 Боец/солдат (soldier)
- ✅ Отметка прибытия в расположение
- ✅ Отметка убытия из расположения
- ✅ Просмотр своего статуса
- ✅ Просмотр общей сводки
- ❌ Доступ к админ-панели

### 👑 Админ (admin)
- ✅ Все права бойца
- ✅ Управление личным составом
- ✅ Просмотр журналов
- ✅ Экспорт данных
- ✅ Настройки уведомлений
- ❌ Управление другими админами

### 👑 Главный админ (main_admin)
- ✅ Все права админа
- ✅ Управление админами
- ✅ Опасные операции
- ✅ Системные настройки
- ✅ Мониторинг бота

## 🗄️ База данных

### Таблица users
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    telegram_id INTEGER UNIQUE NOT NULL,
    name TEXT NOT NULL,
    location TEXT,
    role TEXT DEFAULT 'soldier' CHECK (role IN ('soldier', 'admin', 'main_admin')),
    can_get_notifications BOOLEAN DEFAULT FALSE,
    show_in_reports BOOLEAN DEFAULT TRUE,
    status TEXT DEFAULT 'absent',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### Таблица attendance
```sql
CREATE TABLE attendance (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    date DATE NOT NULL,
    time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    location TEXT,
    status TEXT DEFAULT 'present',
    note TEXT,
    FOREIGN KEY (user_id) REFERENCES users (id),
    UNIQUE(user_id, date)
)
```

## 🔔 Уведомления

### Типы уведомлений
- **Прибытие**: уведомление командирам о прибытии бойца
- **Убытие**: уведомление командирам об убытии бойца
- **Ежедневная сводка**: в 19:00
- **Напоминания**: в 20:30 для неотметившихся

### Получатели
- Все пользователи с ролью `admin` или `main_admin`
- Пользователи с включенными уведомлениями

## 🛠️ Установка и запуск

### 1. Клонирование
```bash
git clone <repository>
cd military-telegram-bot
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 3. Настройка конфигурации
```bash
cp .env.example .env
# Отредактируйте .env файл
```

### 4. Миграция базы данных
```bash
python3 migrate_roles.py
```

### 5. Запуск бота
```bash
python3 main.py
```

# 🔧 Исправление ошибок при запуске

## Проблема
При запуске `1_setup.py` возникала ошибка:
```
bin/sh 1: source not found
```

## Причина
Команда `source` не является стандартной POSIX-командой и не работает в `/bin/sh`. Она специфична для bash. Когда `subprocess.run()` выполняется с `shell=True`, он может использовать `/bin/sh` по умолчанию.

## Решение
1. **Заменены все `source` на `.` (точка)** - это POSIX-стандартная команда для активации скриптов
2. **Улучшена функция `run_command()`** - теперь она явно использует bash для команд с активацией виртуального окружения
3. **Исправлены файлы:**
   - `1_setup.py` - исправлены 4 места с `source`
   - `2_start.sh` - исправлено 1 место с `source`

## Правильный порядок запуска

### 1. Первоначальная настройка
```bash
python3 1_setup.py
```

### 2. Запуск бота
```bash
bash 2_start.sh
```

### 3. Автоматический перезапуск (опционально)
```bash
python3 3_auto_restart.py
```

## Что делает каждый скрипт

### 1_setup.py
- Создает виртуальное окружение Python 3.10
- Обновляет pip
- Устанавливает зависимости из requirements.txt
- Создает базу данных
- Создает скрипт запуска

### 2_start.sh
- Активирует виртуальное окружение
- Проверяет наличие .env файла
- Создает необходимые папки
- Проверяет базу данных
- Запускает бота

### 3_auto_restart.py
- Автоматически перезапускает бота при сбоях
- Мониторит работу процесса
- Ведет логи перезапусков

## Примечания
- Убедитесь, что Python 3.10 установлен на системе
- Создайте файл `.env` с настройками бота перед запуском
- Для PythonAnywhere измените путь в скриптах с `/home/yourusername/mysite` на ваш реальный путь

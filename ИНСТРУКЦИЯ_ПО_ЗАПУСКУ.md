## 📝 Что нужно сделать:

### 1. Настроить `.env` файл

Отредактируйте файл `.env` и замените:
```bash
BOT_TOKEN=ВСТАВЬТЕ_ВАШ_ТОКЕН_БОТА_СЮДА
ADMIN_IDS=ВСТАВЬТЕ_ВАШ_TELEGRAM_ID_СЮДА
```

На реальные значения:
```bash
BOT_TOKEN=1234567890:AABBCCDDEEFFGGHHIIJJKKLLMMNNOOPPQQRRss
ADMIN_IDS=123456789
```

### 2. Как получить токен бота:
1. Найдите в Telegram: @BotFather
2. Отправьте: `/newbot`
3. Укажите имя бота
4. Получите токен

### 3. Как получить свой Telegram ID:
1. Найдите в Telegram: @userinfobot
2. Отправьте: `/start`
3. Получите ваш ID

## 🚀 Способы запуска:

### Способ 1: Автоматический запуск (рекомендуется)
```bash
./start_daily.sh
```

### Способ 2: Ручной запуск
```bash
# Активируем виртуальное окружение
source venv_bot/bin/activate

# Запускаем бота
python start.py
```

### Способ 3: Альтернативные файлы
```bash
source venv_bot/bin/activate
python run_bot.py
# или
python main.py
```

## 🔧 Диагностика проблем:

### Если бот не запускается:
1. Проверьте файл `.env`:
   ```bash
   cat .env
   ```

2. Проверьте виртуальное окружение:
   ```bash
   ls -la venv_bot/
   ```

3. Проверьте зависимости:
   ```bash
   source venv_bot/bin/activate
   pip list
   ```

### Если ошибка "No module named 'auto_restart'":
Файл `auto_restart.py` уже создан! Если проблема повторяется, проверьте:
```bash
ls -la auto_restart.py
```

## 🔄 Автоматический перезапуск:

Бот настроен на автоматический перезапуск каждые 55 минут (для PythonAnywhere).

## ⚙️ Настройки в .env:

```bash
# Основные настройки
BOT_TOKEN=ваш_токен
ADMIN_IDS=ваш_id

# Дополнительные настройки (можно не менять)
DB_PATH=data/bot_database.db
LOG_FILE=logs/bot.log
EXPORT_PATH=exports/
AUTO_RESTART_INTERVAL=3300  # 55 минут
```

## 🆘 Поддержка:

Если возникают проблемы:
1. Проверьте логи: `tail -f logs/bot.log`
2. Убедитесь, что токен и ID правильные
3. Проверьте, что бот активен в BotFather

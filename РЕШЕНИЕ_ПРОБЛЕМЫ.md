# 🔧 Решение проблемы: ClientConnectorError при подключении к Telegram API

## 🚨 Описание проблемы

**Ошибка**: `ClientConnectorError: Cannot connect to host api.telegram.org:443 ssl:default`

**Когда возникает**: При попытке запуска Telegram бота

## 🔍 Диагностика

### Шаг 1: Проверка доступности API
- ✅ Telegram API доступен (проверено через curl)
- ✅ SSL соединение работает
- ✅ DNS резолвинг работает

### Шаг 2: Анализ конфигурации
- ❌ Файл `.env` отсутствует
- ❌ Переменная `BOT_TOKEN` не настроена
- ❌ Переменная `ADMIN_IDS` не настроена

## 🎯 Источник проблемы

**Основная причина**: Отсутствует файл `.env` с настройками бота

**Детали**:
1. Код пытается загрузить `BOT_TOKEN` из переменных окружения
2. Если токен не найден или равен `YOUR_BOT_TOKEN`, возникает ошибка
3. aiogram не может создать соединение с API без валидного токена

## 🛠️ Решение

### 1. Создание файла .env
```bash
cp .env.example .env
```

### 2. Настройка BOT_TOKEN
1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Создайте нового бота: `/newbot`
3. Скопируйте токен
4. Замените `YOUR_BOT_TOKEN` в файле `.env` на реальный токен

### 3. Настройка ADMIN_IDS
1. Откройте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Получите свой ID
3. Замените `YOUR_ADMIN_ID` в файле `.env` на ваш ID

### 4. Проверка конфигурации
```bash
python3 check_config.py
```

## 📋 Внесенные изменения

### Файлы:
1. **`.env`** - Создан файл конфигурации
2. **`config.py`** - Улучшена обработка ошибок

4. **`check_config.py`** - Создан скрипт для проверки настроек

### Улучшения:

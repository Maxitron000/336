## 🛠️ Установка и настройка

### 1. Подготовка окружения

```bash
# Клонирование репозитория
git clone <your-repo-url>
cd telegram-personnel-bot

# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка конфигурации

Создайте файл `.env`:

```env
# Основные настройки
BOT_TOKEN=ваш_токен_бота_от_botfather
ADMIN_IDS=ваш_telegram_id,другой_админ_id
TIMEZONE=Europe/Kaliningrad

# База данных
DB_PATH=data/bot_database.db

# Уведомления
NOTIFICATIONS_ENABLED=true
DAILY_SUMMARY_TIME=19:00
REMINDERS_TIME=20:30

# Логирование
LOG_LEVEL=INFO
LOG_FILE=logs/bot.log

# Экспорт
EXPORT_PATH=exports/
```

### 3. Создание необходимых директорий

```bash
mkdir -p data exports logs config backups
```

### 4. Инициализация базы данных

```bash
python3 -c "
import asyncio
from database import Database

async def init_db():
    db = Database()
    await db.init()
    print('✅ База данных инициализирована')

asyncio.run(init_db())
"
```

## 🚀 Запуск бота

### Локальный запуск

```bash
# Простой запуск
python3 main.py

# Или через скрипт запуска
python3 run_bot.py

# Или через bash скрипт
chmod +x start_bot.sh
./start_bot.sh
```

### Запуск на сервере

#### Systemd (Linux)

Создайте файл `/etc/systemd/system/telegram-bot.service`:

```ini
[Unit]
Description=Telegram Personnel Bot
After=network.target

[Service]
Type=simple
User=your_username
WorkingDirectory=/path/to/your/bot
Environment=PATH=/path/to/your/bot/venv/bin
ExecStart=/path/to/your/bot/venv/bin/python main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# Активация сервиса
sudo systemctl enable telegram-bot
sudo systemctl start telegram-bot
sudo systemctl status telegram-bot
```

#### Screen/Tmux

```bash
# Создание сессии
screen -S telegram-bot

# Запуск бота
python3 main.py

# Отключение от сессии (Ctrl+A, затем D)
# Подключение к сессии
screen -r telegram-bot
```

### 4. Настройка планировщика задач

В разделе "Tasks" на PythonAnywhere добавьте:

**Ежедневное обслуживание (каждый день в 6:00):**
```bash

```

**Проверка здоровья (каждые 4 часа):**
```bash
cd /home/yourusername/your-bot-directory && python3 pythonanywhere_tasks.py health
```

### 5. Настройка Always-on задачи

В разделе "Always-on tasks" создайте задачу:

```bash
cd /home/yourusername/your-bot-directory && python3 main.py
```

### 6. Настройка веб-приложения (опционально)

Для мониторинга через веб-интерфейс создайте WSGI файл:

## 🆘 Устранение неполадок

### Частые проблемы:

#### Бот не отвечает
1. Проверьте логи: `tail -f logs/bot.log`
2. Проверьте статус процесса
3. Выполните принудительный перезапуск через админ-панель

#### Ошибки базы данных
1. Проверьте права доступа к файлу БД
2. Выполните восстановление из резервной копии
3. Проверьте свободное место на диске

#### Проблемы с уведомлениями
1. Проверьте настройки уведомлений в админ-панели
2. Убедитесь, что время на сервере корректное
3. Проверьте права доступа к файлам конфигурации

### Команды диагностики:

```bash
# Проверка статуса бота
python3 -c "from monitoring import BotHealthMonitor; import asyncio; asyncio.run(BotHealthMonitor(None).perform_health_check())"

# Проверка базы данных
sqlite3 data/bot_database.db "SELECT COUNT(*) FROM users;"

# Просмотр логов
tail -f logs/bot.log

# Проверка свободного места
df -h
```

## 🔄 Обновления

### Процесс обновления:

1. **Создайте резервную копию:**
   ```bash
   # Через админ-панель или вручную
   python3 -c "from backup import BackupSystem; import asyncio; asyncio.run(BackupSystem().create_backup('update'))"
   ```

2. **Обновите код:**
   ```bash
   git pull origin main
   pip install -r requirements.txt
   ```

3. **Перезапустите бота:**
   ```bash

   sudo systemctl restart telegram-bot
   ```

4. **Проверьте работоспособность:**
   - Выполните диагностику через админ-панель

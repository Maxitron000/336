# 🚨 Быстрое решение ошибки подключения к Telegram API

## Проблема
```
ClientConnectorError: Cannot connect to host api.telegram.org:443 ssl:default
```

## Причины
❌ Несовместимость aiogram 2.x с кодом для aiogram 3.x  
❌ Не настроен файл `.env` с токеном бота  
❌ Проблемы с SSL в Python 3.13  
❌ Отсутствие виртуального окружения  

## 🚀 Быстрое решение (30 секунд)

### 1. Автоматическая настройка
```bash
python3 setup_venv.py
```

### 2. Получите токен бота
- Откройте [@BotFather](https://t.me/BotFather)
- Отправьте `/newbot`
- Скопируйте токен

### 3. Получите свой ID
- Откройте [@userinfobot](https://t.me/userinfobot)
- Скопируйте ваш ID

### 4. Настройте .env файл
Отредактируйте файл `.env`:
```
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=ваш_telegram_id
```

### 5. Проверьте настройки
```bash
source venv_bot/bin/activate  # Linux/Mac
python3 check_config.py
```

### 6. Запустите бота
```bash
python3 main.py
```

## 🛠️ Ручное решение

### Если автоматическая настройка не работает:

1. **Создайте виртуальное окружение**:
   ```bash
   python3 -m venv venv_bot
   source venv_bot/bin/activate
   ```

2. **Установите зависимости**:
   ```bash
   pip install --upgrade pip
   pip install -r requirements.txt
   ```

3. **Настройте конфигурацию**:
   ```bash
   cp .env.example .env
   # Отредактируйте .env с вашими данными
   ```

## 🚀 Пошаговое решение

### Шаг 1: Получите новый токен от @BotFather

1. Откройте Telegram и найдите **@BotFather**
2. Отправьте команду `/mybots`
3. Выберите вашего бота из списка
4. Нажмите **"API Token"**
5. Скопируйте **полный токен** (должен быть примерно 45-50 символов)

**Если бот был удален:**
1. Создайте нового бота: `/newbot`
2. Введите название бота
3. Введите username (должен заканчиваться на `bot`)
4. Получите токен

### Шаг 2: Проверьте новый токен

Используйте специальный скрипт для проверки токена:

```bash
# Активируйте виртуальное окружение
source venv_bot/bin/activate

# Проверьте токен (замените YOUR_NEW_TOKEN на реальный токен)
python validate_token.py YOUR_NEW_TOKEN
```

**Или интерактивно:**
```bash
python validate_token.py
# Введите токен когда будет запрос
```

### Шаг 3: Обновите конфигурацию

Если токен прошел проверку, обновите файл `.env`:

```bash
# Откройте файл для редактирования
nano .env

# Замените строку:

# На новый токен:

```

### Шаг 4: Проверьте подключение


## 🔍 Формат правильного токена

Токен должен выглядеть примерно так:
```
1234567890:ABCDEF1234567890abcdef1234567890ABC
```

**Характеристики правильного токена:**
- Содержит символ `:` 
- Длина около 45-50 символов
- Начинается с цифр (ID бота)
- После `:` идет длинная строка из букв, цифр и символов

## ⚠️ Возможные проблемы

### Токен не работает:
- Проверьте, что скопировали токен полностью
- Убедитесь, что нет лишних пробелов
- Проверьте, что бот не был удален в @BotFather

### Бот не отвечает:
- Убедитесь, что бот активен в @BotFather
- Проверьте ADMIN_IDS в файле .env
- Посмотрите логи: `tail -f logs/bot.log`

# 🤖 Бот учета личного состава v2.0

Оптимизированный Telegram бот для учета личного состава с админ-панелью, системой уведомлений и поддержкой 24/7 работы на PythonAnywhere.

## ✨ Особенности

### �️ Пользовательский интерфейс
- ✅ Чистый чат: команда `/start` автоматически удаляется
- ✅ Честность интерфейса: тексты соответствуют кнопкам
- ✅ Логичная последовательность действий
- ✅ Максимум эмодзи для лучшего UX

### �️ Админ-панель (многоуровневая структура)
- **Уровень 1: 🏠 Главное меню**
  - � Быстрая сводка с группировкой по локациям
  - 👥 Управление личным составом
  - 📖 Журнал событий
  - ⚙️ Настройки

- **Уровень 2: Меню управления**
  - ✏️ Смена ФИО бойца
  - ➕ Добавление нового бойца
  - ❌ Удаление бойца
  - 📥 Экспорт журнала (CSV/Excel)
  - 🔔 Настройки уведомлений
  - 👑 Управление админами

- **Уровень 3: Опасная зона**
  - 🚨 Отметить всех прибывшими (с подтверждением)
  - 🗑️ Очистка данных
  - 🔄 Сброс настроек

### 🔔 Система уведомлений
- 📊 Ежедневная сводка в 19:00
- ⏰ Напоминания в 20:30
- 🎲 50+ креативных текстов для напоминаний
- 🔇 Тихий режим
- ⚙️ Гибкие настройки

### �️ Хранение данных
- 📁 Отдельный файл `notifications.json` для текстов

## 🚀 Развертывание на PythonAnywhere

### 1. Подготовка

1. **Создайте аккаунт на PythonAnywhere**
   - Зарегистрируйтесь на [pythonanywhere.com](https://www.pythonanywhere.com)
   - Выберите бесплатный план

2. **Создайте Telegram бота**
   - Напишите [@BotFather](https://t.me/BotFather)
   - Получите токен бота

### 2. Загрузка кода

1. **Откройте Bash консоль на PythonAnywhere**
2. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/personnel-bot.git
   cd personnel-bot
   ```

3. **Или загрузите файлы вручную:**
   - Создайте папку `personnel-bot`
   - Загрузите все файлы проекта

### 3. Настройка окружения

1. **Создайте виртуальное окружение:**
   ```bash
   mkvirtualenv --python=/usr/bin/python3.10 personnel-bot
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Создайте файл конфигурации:**
   ```bash
   cp .env.example .env
   ```

4. **Отредактируйте `.env`:**
   ```env
   BOT_TOKEN=your_actual_bot_token_here
   ADMIN_IDS=123456789,987654321
   ```

### 4. Настройка базы данных

1. **Создайте необходимые директории:**
   ```bash
   mkdir -p data exports logs config
   ```

2. **Инициализируйте базу данных:**
   ```bash
   python -c "from database import Database; import asyncio; asyncio.run(Database().init())"
   ```

### 5. Запуск бота

#### Вариант 1: Запуск через консоль
```bash
python run_bot.py
```

#### Вариант 2: Запуск через Always-on task (рекомендуется)

1. **Откройте раздел "Tasks" на PythonAnywhere**
2. **Создайте новую задачу:**
   ```bash
   cd ~/personnel-bot && python run_bot.py
   ```
3. **Установите расписание:**
   - **Daily at:** 05:00
   - **Command:** `cd ~/personnel-bot && python run_bot.py`

### 6. Настройка веб-приложения (опционально)

1. **Создайте веб-приложение на PythonAnywhere**
2. **Укажите путь к проекту:** `/home/yourusername/personnel-bot`
3. **Настройте WSGI файл для мониторинга**

## �️ Устранение неполадок

### Бот не отвечает
1. Проверьте логи: `tail -f bot.log`
2. Убедитесь, что токен правильный
3. Проверьте права администратора

### Ошибки базы данных
1. Проверьте права доступа к папке `data/`
2. Пересоздайте базу: удалите `bot_database.db`
3. Запустите инициализацию заново

### Проблемы с уведомлениями
1. Проверьте настройки в `.env`
2. Убедитесь, что ID админов правильные
3. Проверьте файл `notifications.json`

## � Поддержка

При возникновении проблем:
1. Проверьте логи в файле `bot.log`
2. Убедитесь, что все зависимости установлены
3. Проверьте конфигурацию в `.env`

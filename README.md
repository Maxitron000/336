## 🗄️ База данных

Система использует SQLite с следующими таблицами:

### `users` - Пользователи
- `id` - Уникальный идентификатор
- `telegram_id` - Telegram ID
- `full_name` - ФИО в формате "Фамилия И.О."
- `username` - Telegram username
- `is_admin` - Статус администратора
- `registered_at` - Дата регистрации
- `last_activity` - Последняя активность

### `location_logs` - Журнал местоположений
- `id` - Уникальный идентификатор
- `user_id` - ID пользователя
- `location` - Локация
- `action` - Действие (arrived/left)
- `timestamp` - Время действия

### `active_sessions` - Активные сессии
- `id` - Уникальный идентификатор
- `user_id` - ID пользователя
- `location` - Текущая локация
- `entered_at` - Время входа

### `admin_logs` - Административные логи
- `id` - Уникальный идентификатор
- `admin_id` - ID администратора
- `action` - Выполненное действие
- `target_user_id` - ID целевого пользователя
- `details` - Детали операции
- `timestamp` - Время операции

## 🚨 Решение проблем

### Частые проблемы:

1. **Бот не запускается**
   - Проверьте токен в `.env` файле
   - Убедитесь, что установлены все зависимости

2. **Пользователь не может зарегистрироваться**
   - Проверьте формат ФИО: "Фамилия И.О."
   - ✅ Правильно: "Иванов И.И.", "Петров П.П.", "Сидоров С.С."
   - ❌ Неправильно: "Иванов Иван", "И.И. Иванов", "иванов и.и."
   - Убедитесь, что пользователь не заблокирован

3. **Не работают уведомления**
   - Проверьте настройку `ENABLE_NOTIFICATIONS=true`
   - Убедитесь в корректности времени уведомлений

4. **Ошибки базы данных**
   - Используйте `check_database_integrity()` для проверки
   - Восстановите из резервной копии при необходимости

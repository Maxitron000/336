# Отчет о найденных и исправленных багах

## Баг #1: Отсутствие проверки существования директории для логов

### Описание проблемы
В файле `main.py` при настройке логирования не проверяется существование директории `logs/`. При первом запуске бота это приведет к ошибке `FileNotFoundError`.

### Последствия
- Бот не сможет запуститься при первом запуске
- Потеря логов и невозможность отладки

### Исправление
```python
# Создаем директорию для логов если её нет
os.makedirs('logs', exist_ok=True)
```

## Баг #2: Несовместимость timezone-aware и timezone-naive datetime

### Описание проблемы
В файле `notifications.py` функция `get_current_time()` возвращает datetime с часовым поясом (timezone-aware), но `datetime.combine()` создает naive datetime без часового пояса. При попытке вычитания возникнет `TypeError`.

### Последствия
- Система уведомлений не будет работать
- Ошибки в планировщике задач
- Неправильное время отправки уведомлений

### Исправление
```python
# Добавляем часовой пояс к target_time
target_time = now.tzinfo.localize(target_time)
```

## Баг #3: Блокирующие операции в асинхронном коде

### Описание проблемы
В файле `export.py` используются синхронные операции записи файлов (`with open()`) внутри асинхронных функций. Это блокирует event loop и может привести к зависанию бота при больших объемах данных.

### Последствия
- Замедление работы всего бота
- Невозможность обработки других команд во время экспорта
- Таймауты при работе с большими файлами

### Исправление
```python
# Формируем CSV в памяти
output = io.StringIO()
writer = csv.DictWriter(output, fieldnames=fieldnames)
writer.writeheader()
writer.writerows(export_data)

# Записываем асинхронно
await asyncio.get_event_loop().run_in_executor(
    None, 
    lambda: open(filepath, 'w', encoding='utf-8').write(csv_content)
)
```

## Баг #4: Критическая уязвимость безопасности

### Описание проблемы
В файле `config.py` токен бота хранится в открытом виде прямо в коде. Это критическая проблема безопасности, так как токен может быть скомпрометирован при публикации кода.

### Последствия
- Несанкционированный доступ к боту
- Возможность отправки спама от имени бота
- Кража персональных данных пользователей
- Необходимость смены токена после компрометации

### Исправление
```python
# Загружаем токен из переменных окружения
TOKEN = os.getenv('BOT_TOKEN')
if not TOKEN:
    raise ValueError("BOT_TOKEN не найден в переменных окружения!")
```

## Рекомендации

1. **Немедленно смените токен бота** через @BotFather, так как текущий токен скомпрометирован
2. Добавьте `.env` в `.gitignore` чтобы не допустить повторной утечки
3. Используйте переменные окружения для всех чувствительных данных
4. Регулярно проверяйте код на наличие хардкода секретов
5. Настройте CI/CD для автоматической проверки безопасности

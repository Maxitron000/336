### 🛠️ Новые команды

#### `/health` - Проверка здоровья бота
- Отправляет подробный отчет о состоянии системы главному админу
- Проверяет подключение к БД, доступность бота, статистику пользователей

#### `/performance` - Отчет производительности
- Анализирует эффективность работы системы
- Показывает процент присутствия и категорию производительности

#### `/maintenance [тип] [длительность]` - Уведомление о ТО
- Отправляет уведомление о техническом обслуживании
- Поддерживает различные типы ТО с указанием длительности

### 🎛️ Обновленная админ-панель

#### Новые разделы:
- **🏥 Мониторинг бота** - полный контроль состояния системы
- **🔧 Техобслуживание** - управление обслуживанием

#### Новые функции в админ-панели:
- **Проверка здоровья** - отправка отчета главному админу
- **Отчет производительности** - отправка отчета главному админу
- **Статистика системы** - локальная статистика
- **Детальная диагностика** - полная проверка компонентов
- **История статусов** - последние события системы
- **Лог техобслуживания** - история ТО

### 🔧 Технические изменения

#### Файл `notifications.py`
- Добавлены методы:
  - `send_bot_status_notification()` - уведомления о статусе
  - `send_bot_health_check()` - проверка здоровья
  - `send_bot_performance_report()` - отчет производительности
  - `send_bot_error_notification()` - уведомления об ошибках
  - `send_bot_maintenance_notification()` - уведомления о ТО

#### Файл `database.py`
- Добавлен метод `is_connected()` - проверка подключения к БД

#### Файл `main.py`
- Обновлены функции `on_startup()` и `on_shutdown()` с уведомлениями
- Добавлены обработчики команд `/health`, `/performance`, `/maintenance`
- Добавлены автоматические проверки в планировщик
- Обновлен обработчик ошибок с уведомлениями

#### Файл `admin.py`
- Добавлены методы мониторинга:
  - `get_system_stats()` - статистика системы
  - `format_system_stats()` - форматирование статистики
  - `get_detailed_diagnostics()` - детальная диагностика
  - `get_status_history()` - история статусов
  - `perform_maintenance()` - выполнение ТО
  - `get_maintenance_log()` - лог ТО
- Обновлен конструктор для поддержки системы уведомлений
- Добавлена обработка новых callback'ов для мониторинга и ТО

#### Файл `keyboards.py`
- Добавлены новые клавиатуры:
  - `get_monitoring_keyboard()` - мониторинг бота
  - `get_maintenance_keyboard()` - техобслуживание
  - `get_maintenance_confirmation_keyboard()` - подтверждение ТО
- Обновлена главная админ-клавиатура с новыми разделами

### 📊 Расписание автоматических проверок

- **09:00** - Ежедневная проверка здоровья бота
- **19:00** - Ежедневная сводка (существующая)
- **20:30** - Напоминания (существующая)
- **Воскресенье 10:00** - Еженедельный отчет производительности

### 🛡️ Безопасность

- Уведомления о статусе отправляются только главному админу
- Уведомления об ошибках отправляются всем админам
- Все действия логируются в базу данных
- Подтверждение требуется для операций ТО
